Storing the data

There are 3 main types of data stored in the system:

  • raw data: various formats stored as individual files

  • gridded data: MDV data stored as one file per data time

  • non-gridded data: SPDB data stored in daily files

Data directory

The main data directory is: 

    ~/projDir/data

All data is stored in subdirectories of the main directory.

Raw data

This is stored in: 

    ~/projDir/data/raw

Generally the raw data files will be named in a way which conveys the date and time of the data. Examples are:

AFTN data for 10:35 UTC on 2002/11/14: 

  ~/projDir/data/raw/aftn/20021114/103500.aftn

MM5 model output data for 7-hour forcast from model run at 00 UTC on 2002/11/15: 

  ~/projDir/data/raw/mm5/domain3/d3.2002111500.tm0700.mm5

MDV data

MDV stands for Meteorological Data Volume. It is the data format used in the system for regular grids, such as radar, satellite and model data.

MDV data is stored in: 

    ~/projDir/data/mdv

The MDV files are named based on their data time. There are two naming conventions, one for observational data such as radar data, and the other for forecast data, such as model output. Examples are:

Domain3 satellite data for 16:15:14 UTC on 2002/11/13: 

    ~/projDir/data/mdv/sat/domain3/20021113/162514.mdv

2-hour (7200 sec) forecast for domain3 MM5 model data generated by the model run at 09:00 UTC on 2002/11/13: 

    ~/projDir/data/mdv/mm5/domain2/20021113/g_090000/f_00007200.mdv




SPDB data

SPDB stands for Symbolic Product Data Base. SPDB is used for storing
all non-gridded data, such as METARs, TAFs, lightning strikes etc.


SPDB data is stored in:


    ~/projDir/data/spdb



Since this is a data base, there is not a single file per data observation.
Instead, there is one pair of files per day, one index (header) file and
one data file. For example, the file:


    ~/projDir/data/spdb/ltg/20021113.indx


    ~/projDir/data/spdb/ltg/20021113.data



contain the lightning data for 2002/11/13.
